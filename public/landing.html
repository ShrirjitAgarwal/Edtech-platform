How a school gets running

A guided path that takes an institution from agreement to a student's first submission.

1

Onboard the school

Your school is created and an administrator account is set up and handed over securely.

2

Set up classes

The admin adds subjects, classes, teachers, and students, then maps who teaches what.

3

Build assessments

Teachers create MCQ and coding tests, set duration and schedule, then publish to a class.

4

Students sit the test

Students enter with the school code, take their assigned test, and results post to reports.

Trust & data protection

Student data, handled with care

Assessment means handling minors' personal data. The platform is built so that protection is structural, not an afterthought.

School isolation

Every school's data is scoped at the server. No school can access another's students, tests, or results.

Role-based access

Admin, teacher, and student each reach only what their role allows — enforced server-side on every request.

Audit logging

Sensitive actions like report downloads are logged with a user record and timestamp.

Data terms at sign-up

Schools accept data processing terms at onboarding and acceptance is recorded.
